2. Problem Statement & Motivation

Generative AI image tools (Midjourney, DALL-E, Stable Diffusion) have exploded in use, but they demand precise, structured prompts to produce high-quality results. Two problems exist in this space:

  • Reverse engineering: Given an AI-generated image, there is no reliable way to recover the prompt that created it. Existing tools simply ask an LLM to guess — and LLMs hallucinate physical properties freely.
  • Prompt improvement: Non-expert users write vague, low-quality prompts. There is no easy mobile interface to iteratively refine them using AI assistance.

RePrompt addresses both problems. The web application solves reverse engineering by grounding Gemini’s output in objective computer vision measurements — making the recovered prompt factually anchored to the pixel data. The Android application solves prompt improvement by providing a clean, dark-themed mobile studio for iterative AI-assisted prompt crafting.

3.2 Web Application Pipeline

Every image uploaded to the web app travels through six distinct stages before a prompt is returned. The pipeline is fully modular — each stage has a defined input/output contract and can be tested independently.

Stage Component What Happens
1 Browser (index.html + script.js) The user uploads an image via drag-and-drop or file picker. FileReader renders local preview instantly — zero server round-trip. Image POSTed to /api/reprompt as multipart/form-data.
2 FastAPI (main.py) Receives upload → writes secure OS tempfile → calls preprocess() → analyze_physics() → generate_prompt_from_gemini(). Tempfile deleted in finally block regardless of outcome.
3 Preprocessor Validates format, letterbox-resizes to 1024×1024 (preserving aspect ratio), applies bilateral denoise, computes grayscale stats, runs HSV colour histogram. Returns PreprocessMetadata with live numpy arrays.
4 Physics Engine Runs 5 CV algorithms: Laplacian variance (DoF), Sobel gradients (shadow hardness), brightness centroid (light direction), Canny edge ratio (composition density), P95/P5 contrast ratio.
5 Gemini API Sends base64 image + physics metadata to Gemini. 4-model fallback: gemini-2.5-flash-lite → gemini-2.0-flash-lite → gemini-2.5-flash → gemini-2.0-flash. On 429, waits 5s and retries before advancing.
6 UI Renders prompt and physics stat cards dynamically. Copy-to-clipboard via navigator.clipboard. PDF export via jsPDF.

4. Generative AI Integration

4.1 The Core Innovation: Grounded Generation

Why hybrid CV + LLM? Pure LLM prompting hallucinates measurements. Gemini might say “soft lighting” when the Sobel gradient score objectively shows hard shadows at 25.5. The OpenCV layer measures real pixel physics first — Gemini is then constrained to those verified numbers. The result is dramatically more reproducible and factually accurate.

The web application implements a two-phase AI approach that differentiates RePrompt from simple LLM image-description tools: * Phase 1 — Computer Vision (deterministic): OpenCV algorithms extract five objective measurements from the raw pixel data. These measurements are invariant — the same image always produces the same numbers. * Phase 2 — Language Model (constrained): Gemini receives the image AND the verified measurements as structured context. The prompt engineering template explicitly tells Gemini to treat the CV numbers as ground truth and build its narrative around them.

4.2 Physics Measurements (Web App)

Measurement Algorithm Output & Thresholds
Brightness Class Mean pixel value of grayscale “high-key” (mean > 170) / “mid-key” / “low-key” (mean < 85)
Depth of Field Laplacian variance — cv2.Laplacian(gray, CV_64F).var() “deep” (>300) / “moderate” / “shallow” (<80). High variance = many sharp edges.
Shadow Hardness Sobel gradient mean magnitude — sqrt(Sx² + Sy²) “hard” (>12.0) / “soft” / “diffuse” (<5.0). Sharp edges = point light source.
Light Direction Moment centroid of brightest 10% pixels (cv2.moments) Compass label: top-left, top-right, left, right, flat. Normalised to [-1, 1].
Edge Density Canny edge pixel count / total pixels 0.0–1.0 ratio. High = busy/detailed. Low = minimalist/clean.
Contrast Ratio P95 luminance / (P5 luminance + 1e-5) Robust ratio avoiding clipping noise. E.g. 25.5 = high drama.

4.3 Gemini Model Fallback Strategy

Both products implement a resilient multi-model fallback pattern to maximise uptime under free-tier quota limits:

Priority Model Trigger Condition Free Quota
1st (primary) gemini-2.5-flash-lite Always tried first Highest — lite models have more headroom
2nd gemini-2.0-flash-lite 429 rate-limit on model 1 after retry Independent quota pool
3rd gemini-2.5-flash 429 on model 2 Independent quota pool
4th gemini-2.0-flash 429 on model 3 Independent quota pool

On each 429 response, the system waits 5 seconds and retries once before advancing to the next model. This gives effectively 4× the free-tier headroom compared to a single-model implementation.

4.4 Cloud Architecture Transition (Updated)

We have moved away from the initial local ngrok setup for the Android backend. The system now utilizes a persistent, cloud-based Render deployment (https://reprompt-backend-0qmw.onrender.com/improve), ensuring 24/7 availability for the mobile application without URL rotation issues.

5. Web Application — Detailed Breakdown

5.1 Backend: main.py

Central entry point. Wires together FastAPI, both CV modules, and the Gemini API into one coherent service. Serves both the REST API and static frontend from a single Uvicorn process. The API key is securely loaded from the GEMINI_API_KEY environment variable.

Function / Route Role Key Detail
_gemini_post(payload) Resilient Gemini caller Iterates GEMINI_MODELS in order. On 429, waits 5s and retries once before falling to next model.
generate_prompt_from_gemini() Vision synthesis Reads tempfile → base64 → structured prompt with CV stats injected → calls _gemini_post() via asyncio.run_in_executor.
POST /api/reprompt Main pipeline Image upload → preprocess() → analyze_physics() → Gemini → JSON {reprompt, stats}. Tempfile in finally block.
GET /improve Text enhancer Takes raw prompt string, enhances via Gemini, returns improved version. Stateless and highly available.
app.mount("/") Static serving Mounts static/ as frontend. One Uvicorn process handles both API and UI — no separate Nginx.

5.2 Image Preprocessor: modules/preprocessors.py

Standardises every image to a clean 1024×1024 numpy array. Downstream modules depend on this contract — they assume a fixed-size denoised image with known padding offsets.

Function Algorithm Why This Approach
validate_image() Extension allowlist + cv2.imread decode Fails loudly before wasting compute. Rejects unsupported formats early.
letterbox_resize() Scale longest axis to 1024, black-pad short axis Preserves true geometry. Stretch distorts the brightness centroid, giving wrong light direction.
reduce_noise() cv2.bilateralFilter (d=9, σ=75) Edge-preserving denoise. Gaussian blur corrupts edge measurements used by Sobel and Laplacian.
analyze_color_histogram() HSV histogram, 36 bins, top-3 peaks Provides Gemini with verified hue data — LLM colour perception is inconsistent across model versions.

5.3 Frontend: static/script.js

All interactivity, API communication, animations and PDF export. Zero framework dependencies — plain ES6 JavaScript. Uses IntersectionObserver for scroll reveals and jsPDF for client-side PDF generation.

6.3 Networking Layer

The networking layer has been completely refactored to eliminate third-party dependencies like OkHttp, now utilizing native HttpURLConnection for lightweight and resilient communication. Key characteristics include:

  • Native GET Requests: Communicates directly with the persistent Render backend (https://reprompt-backend-0qmw.onrender.com/improve) using URL-encoded parameters.
  • Extended Resilience: Timeouts have been increased to 120 seconds to robustly handle the realities of AI inference latency and potential cold-starts on the cloud.
  • UI Thread Isolation: Network operations are executed asynchronously, preventing any main thread blocking while the “Backend is thinking…” progress state is active.

7. Key Design Decisions

Decision Rationale
FastAPI over Flask FastAPI is async-native. The pipeline is I/O-bound (waiting for Gemini HTTP). asyncio + run_in_executor offloads the blocking HTTP call to a thread pool, keeping the event loop free for concurrent requests.
Model Verification A new verify_and_start() function dynamically checks for available Gemini models on the backend, prioritizing gemini-1.5-flash for the best balance of speed and quality.
API Key Security The GEMINI_API_KEY has been strictly moved to an environment variable accessed via os.environ.get, adhering to deployment security best practices and preventing credential leaks.
Hybrid CV + LLM Pure LLM hallucinates physics measurements. OpenCV measures real pixel data first; Gemini is constrained to those verified numbers. This is the core value proposition of RePrompt.
Bilateral filter Unlike Gaussian blur, bilateral preserves edge sharpness while smoothing flat areas. Critical because Sobel and Laplacian both measure edge properties — blurring them would corrupt results.
Stateless web backend Analysis tool, not a history service. The single /improve endpoint remains stateless. Tempfiles deleted per-request. Zero storage cost, zero privacy risk. Compatible with Render’s free ephemeral disk.
Vanilla JS frontend React/Vue adds 200KB+ bundle overhead for a 3-tab SPA. Plain ES6 with IntersectionObserver delivers smooth animations at near-zero cost.
Native HttpURLConnection Replaced OkHttp with native HttpURLConnection and URL-encoded GET parameters. This reduces the Android app’s dependency footprint while reliably connecting to the persistent Render backend.
Cloud Hosting over ngrok Transitioning from local ngrok to a persistent cloud backend (Render) eliminates the “ngrok URL rotation” issue, providing a stable, highly available endpoint for the mobile app.
Markdown Rendering Integrated the Markwon library (io.noties.markwon.Markwon) to handle Markdown formatting in the output TextView, ensuring the AI’s generated prompts are displayed with production-ready professionalism.
